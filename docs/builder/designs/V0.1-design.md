# EvoGenesis V0.1 / V0.1.1 详细设计：RNA 世界

## 1. 阶段目标

`v0.1` 已完成 RNA 世界基础模拟：RNA 家族复制、突变、资源竞争、降解、谱系兴衰、事件日志和快照恢复。

用户反馈表明，下一步不应急着进入 `v0.2 前生命化学`，而应先做 `v0.1.1 运行体验修正版`。本修正版不扩展生命史阶段，只把当前基础模拟调整为更符合“演化世界”的体验：默认持续演化、可暂停、可重置、可调速、中文可读、事件更具体、运行结束有总结。

本阶段仍禁止实现前生命化学、原始细胞、原核生命、光合作用、氧气事件或宏观生态。

## 2. 项目结构与职责边界

建议继续保持当前结构，并在不破坏职责边界的前提下增加运行控制与总结能力。

```text
evogenesis/
  cli.py                 命令入口和中文帮助文本
  engine.py              模拟主循环、tick 推进、随机源、暂停保存
  state.py               可序列化世界状态、RNA 家族、事件记录、统计指标
  config.py              配置加载、校验、默认参数
  events.py              事件定义、触发检查、证据字段
  summary.py             运行总结，可新增
  runtime.py             运行控制，可新增
  modules/
    rna_world.py         RNA 世界规则
  io/
    snapshots.py         快照保存与恢复
    history.py           历史日志、指标输出、中文摘要
configs/
  rna_world.default.json
README.md
tests/
```

职责要求：

- 命令入口只负责参数解析、用户交互和中文输出，不直接修改模拟规则。
- 模拟引擎拥有 tick、随机源、调度顺序、运行循环和暂停保存时机。
- RNA 世界规则只改变模拟状态，不直接写日志或控制台。
- 事件系统读取状态和本 tick 变化，生成事件记录，不直接推进规则。
- 输入输出层负责序列化、日志和指标，不决定演化结果。
- 总结系统读取事件、指标和最终状态，生成中文概述，不修改状态。

## 3. 命令与运行语义

### 3.1 当前问题

当前 `--ticks` 表示“本次追加运行多少 tick”。如果从 `saves/latest.json` 恢复，而快照已经在 tick 4000，再传入 `--ticks 1000`，最终会到 tick 5000。用户容易误解为“从 0 跑到 1000”或“从 0 跑到 5000”。

### 3.2 修正方向

短期必须把语义改清楚：

```text
python -m evogenesis run
python -m evogenesis run --steps 1000
python -m evogenesis run --until-tick 5000
python -m evogenesis run --resume saves/latest.json --steps 1000
python -m evogenesis pause
python -m evogenesis reset
python -m evogenesis inspect saves/latest.json
```

建议：

- `run` 默认进入持续运行模式，直到用户暂停或中断。
- `--steps` 表示“本次追加运行多少 tick”，用于测试。
- `--until-tick` 表示“运行到目标 tick”，若当前 tick 已超过目标，应拒绝并提示。
- `--ticks` 若暂时保留，应标记为兼容旧参数，并在帮助中说明等同于 `--steps`。
- 非恢复运行必须从 tick 0 创建新世界。
- 恢复运行必须明确打印“从 tick X 恢复，计划追加 Y tick / 运行到 tick Z”。

### 3.3 暂停

暂停目标是“关闭程序前保存当前世界，下次直接继续”。

基础要求：

- 用户主动暂停时立即保存 `saves/latest.json`。
- 控制台打印中文提示：当前 tick、保存路径、下一次恢复命令。
- 持续运行模式下，键盘中断不应直接丢失状态，应捕获后保存快照再退出。

### 3.4 重置

重置目标是清理测试阶段状态，确保下一个版本或下一轮测试从头开始。

基础要求：

- `reset` 不应误删用户未确认的数据。
- 推荐将旧数据移动到带时间戳的归档目录，或要求明确参数后清理。
- 重置后下一次 `run` 必须从 tick 0 开始。
- 重置范围至少包括 `saves/latest.json`、tick 快照、日志和指标；配置文件不应被删除。

## 4. 时间流速控制

需要区分两个概念：

- 模拟步长：一个 tick 在世界内代表多少时间，例如 1 年、1000 年、10000 年。
- 运行速度：现实时间里每秒推进多少 tick，影响用户观察节奏。

配置建议：

```text
time_unit_label: 年 / 千年 / 万年
years_per_tick: 每 tick 对应的模拟年份
ticks_per_second: 控制台持续运行时每秒推进多少 tick
status_interval_ticks: 每隔多少 tick 打印一次关键参数
snapshot_interval: 每隔多少 tick 保存一次快照
metrics_interval: 每隔多少 tick 写一次指标
```

实现优先级：

1. 先通过配置文件设置基础速率和输出间隔。
2. 控制台运行中暂不强制做复杂多线程改速。
3. 若要改速，用户可以暂停保存，修改配置，再恢复运行。
4. 后续若进入图形界面，再设计运行中调速滑块。

## 5. 中文优先

用户可见文本必须中文优先：

- README 使用中文说明。
- 命令帮助、错误信息、运行摘要、事件消息、最终总结使用中文。
- 控制台中可以保留稳定 ID，但必须同时显示中文名。
- 文档中如保留英文技术词，应给出中文解释。

事件示例：

```text
[tick 4913] 资源崩溃 resource_crash
原因：资源池 0.7 低于阈值 960.0，复制增长受限。
影响：RNA 家族复制速度下降，竞争压力上升。
```

## 6. 控制台观察与日志

当前控制台只输出事件和最终摘要，不足以观察演化过程。修正版要求：

- tick 开始时打印中文启动消息，例如“RNA 世界开始演化”。
- 持续运行时按 `status_interval_ticks` 打印关键参数。
- 关键参数至少包括：tick、模拟时间、总种群、家族数、多样性、资源池、灭绝数、新家族数、优势家族。
- 重要事件发生时，除消息外应显示触发依据。
- `history.log` 可继续用机器友好的结构化格式，但应增加用户友好的中文事件摘要文件。
- `metrics.csv` 继续用于观察趋势；后续图形界面可直接读取。

关于“每个 tick 的所有属性都记录下来”：

- 不建议默认记录所有属性，否则日志量会很快膨胀。
- 推荐默认记录周期性指标，重要事件发生时记录完整证据快照。
- 如用户需要全量追踪，可通过配置开启 `trace_all_ticks`，并明确提示文件会很大。

## 7. 图形界面方向

当前阶段仍以命令行为主，但用户提出的观察需求确实更适合图形界面。建议把图形界面列为中期方向，不阻塞 `v0.1.1`。

候选界面：

- 左侧或顶部：主要参数面板。
- 中央：重要事件流。
- 右侧：选中事件时显示事件发生时的完整参数和证据。
- 可展开区域：全部参数、家族列表、谱系信息。
- 控制区：开始、暂停、重置、速度控制。

约束：

- GUI 不应改变核心模拟规则。
- GUI 只调用引擎、状态、事件和日志接口。
- 在决定 GUI 技术栈前，不引入重量级依赖。

## 8. 状态模型补充

当前状态模型可继续使用，但为了支持运行体验，需要补充或明确：

```text
run_id                    本次运行会话 ID
world_created_at          世界创建时间
last_saved_at             最近保存时间
years_per_tick            模拟时间换算
elapsed_simulation_years  已经历模拟时间
run_mode                  continuous / steps / until_tick
```

RNA 家族事件需要能列出具体对象，因此事件证据中应包含：

```text
family_ids
lineage_ids
family_snapshots
population_before
population_after
```

其中 `family_snapshots` 只记录必要字段，例如家族 ID、谱系 ID、种群、序列、稳定性、催化能力，不需要复制完整世界状态。

## 9. 事件系统修订

事件不是随机打印文本，而是从状态中识别出的模拟结果。事件结构建议：

```text
事件定义 EventDefinition
  id                  稳定事件 ID
  display_name        中文名
  category            里程碑 / 趋势 / 灾难
  era                 所属阶段
  trigger_conditions  触发条件
  probability_model   概率检定
  cooldown_or_once    冷却或只触发一次
  severity            重要程度
  effects             影响
  message_templates   中文消息模板
  evidence_fields     证据字段

事件记录 EventRecord
  id
  display_name
  tick
  category
  message
  evidence
  affected_families
  affected_lineages
  effects
```

`v0.1 RNA 世界` 事件目录应有中文显示名：

```text
first_replicator              首个复制子扩张
stable_replicator_lineage     稳定复制子谱系
mutation_burst                突变爆发
dominant_lineage              优势谱系形成
resource_crash                资源崩溃
replicator_extinction         复制子灭绝
catalytic_breakthrough        催化突破
complexity_increase           复杂度上升
```

必须修正的事件输出：

- “一个谱系现已占据主导”必须列出具体 `lineage_id`、相关家族 ID、占比和种群。
- “一个或多个 RNA 家族退化至灭绝”必须列出本 tick 灭绝的家族 ID，数量过多时列出前若干个并说明总数。
- 事件 evidence 必须能解释为什么发生，而不是只写一句结论。

## 10. 运行总结系统

最终总结不应只是事件堆砌，而应从事件、指标和最终状态中生成中文概述。

总结至少回答：

- 本次模拟从哪个 tick 到哪个 tick。
- RNA 世界是否成功维持复制，还是走向灭绝。
- 出现了哪些关键演化进展。
- 哪些谱系或家族成为优势。
- 资源是否崩溃，多样性是上升还是下降。
- 当前演化发展到哪一步，距离下一阶段还缺什么。

这不需要引入 Agent。可以先用规则化总结器实现：

```text
输入：
- 起始状态
- 结束状态
- 事件列表
- 指标趋势

输出：
- 3 到 6 句中文总结
- 关键事件列表
- 当前阶段判断
- 风险或限制因素
```

后续如果确实需要自然语言更丰富，再考虑模板库或外部模型，但不作为 `v0.1.1` 前提。

## 11. 快照与数据文件

`latest.json` 适合机器恢复，不适合用户阅读。应区分两类文件：

- 机器文件：快照继续使用 JSON，保证恢复准确。
- 用户文件：增加中文摘要、事件摘要和指标表，方便阅读。

建议输出：

```text
saves/latest.json                 机器恢复快照
saves/tick-1000.json              周期快照
logs/history.jsonl                结构化事件日志
logs/events.txt                   中文事件日志
logs/metrics.csv                  指标趋势
logs/latest-summary.txt           最近一次运行总结
```

并发风险：

- 同一目录下不应允许多个模拟同时写 `latest.json`。
- 若检测到运行锁，应拒绝第二个运行进程，或要求指定独立输出目录。

## 12. 测试计划补充

新增测试：

- 非恢复运行从 tick 0 开始。
- 恢复运行明确从快照 tick 继续。
- `--steps` 是追加 tick，`--until-tick` 是目标 tick。
- 当前 tick 大于目标 tick 时，`--until-tick` 拒绝运行。
- 暂停或键盘中断会保存 `latest.json`。
- 重置后下一次运行从 tick 0 开始，旧数据不被误用。
- 中文输出包含启动消息、关键参数、事件中文名和总结。
- 灭绝事件包含具体家族 ID。
- 优势谱系事件包含谱系 ID、家族 ID、种群和占比。
- 总结系统能识别“维持复制 / 资源崩溃 / 灭绝 / 稳定演化”等基本结论。
- 同一输出目录并发运行时不会互相覆盖 `latest.json`。

## 13. 验收标准

`v0.1.1` 完成后应满足：

- 用户能直接启动持续演化，并看到“RNA 世界开始演化”的中文提示。
- 用户能暂停并保存，下次从保存状态继续。
- 用户能重置测试数据，并确认新世界从 tick 0 开始。
- 用户能通过配置控制模拟时间换算和观察速度。
- 控制台持续显示关键参数，而不是只在最后显示摘要。
- 事件消息能列出具体谱系、家族和触发证据。
- 运行结束或暂停时生成中文总结。
- README 和主要用户可见文本中文优先。
- 快照仍可恢复，随机状态连续。
- 不引入未经说明的重量级依赖。
