# EvoGenesis V0.1 详细设计：RNA World

## 1. Stage 目标

`v0.1` 实现一个可运行、可复现、可保存和可检查的 RNA World 命令行模拟器。模拟对象是原始环境中的 RNA 家族，它们围绕资源进行复制、突变、降解和竞争，并通过事件系统记录谱系兴衰。

本阶段禁止实现后续版本的前生命化学、原始细胞、原核生命、光合作用、氧气事件或宏观生态。

## 2. 建议项目结构

```text
evogenesis/
  __init__.py
  cli.py
  engine.py
  state.py
  config.py
  events.py
  modules/
    __init__.py
    rna_world.py
  io/
    __init__.py
    snapshots.py
    history.py
configs/
  rna_world.default.json
tests/
README.md
```

职责边界：

- `cli.py`：命令行入口，只做参数解析和结果展示。
- `engine.py`：模拟主循环，拥有 tick 推进、模块调度、随机源和快照时机。
- `state.py`：定义可序列化状态对象，不直接读写文件。
- `config.py`：加载和校验配置，提供默认参数。
- `modules/rna_world.py`：RNA World 的复制、突变、竞争、降解规则。
- `events.py`：事件定义、触发检查、概率检定和事件记录。
- `io/snapshots.py`：JSON 快照保存与恢复。
- `io/history.py`：事件日志与指标 CSV 输出。

## 3. CLI 范围

首版至少支持：

```text
python -m evogenesis run --ticks 10000
python -m evogenesis run --config configs/rna_world.default.json
python -m evogenesis run --resume saves/latest.json --ticks 5000
python -m evogenesis inspect saves/latest.json
```

失败模式：

- 配置文件不存在或格式错误：给出可读错误，不创建半截输出。
- 快照版本不兼容或字段缺失：拒绝恢复，并说明缺失字段。
- ticks 非正数：拒绝运行。

```text

```

状态所有权：

- Engine 拥有 tick、随机源和调度顺序。
- RNA World 模块可以改变 RNA 家族、资源池和阶段统计。
- Event System 可以读取状态并追加事件记录，但不直接推进模拟规则。
- IO 层只能序列化或输出状态，不能修改模拟结果。

## 5. RNA World 规则

复制：

- RNA 家族按复制率、催化能力、资源可用性产生后代。
- 复制必须消耗资源，资源不足时增长受限。

突变：

- 复制时根据错误率改变序列。
- 发生有效突变时派生新家族，并保留 `parent_id` 与 `lineage_id`。
- 突变可能改变长度、稳定性、复制率、催化能力和资源成本。

降解：

- 稳定性低、环境压力高、资源枯竭时更容易减少或灭绝。
- 灭绝应产生可被事件系统识别的状态变化。

竞争：

- 资源有限，复制效率高但成本高的序列应与短小稳定序列形成动态竞争。
- 不要求真实化学精度，但每个变量应有明确生物学含义。

## 6. V0.1 事件系统

事件不是随机打印文本，而是从状态中识别出的模拟结果。首版应建立轻量结构：

```text
EventDefinition
  id
  name
  category
  era
  trigger_conditions
  probability_model
  cooldown_or_once
  severity
  effects
  message_templates
  evidence_fields

EventRecord
  id
  tick
  category
  message
  evidence
  effects
```

事件分类：

- `Milestone`：关键阶段达成。
- `Trend`：状态持续变化。
- `Disaster`：本阶段可暂不实现自然灾难，但结构上允许未来加入。

`v0.1 RNA World` 事件目录：

```text
first_replicator
stable_replicator_lineage
mutation_burst
dominant_lineage
resource_crash
replicator_extinction
catalytic_breakthrough
complexity_increase
```

事件触发原则：

- 每个事件必须有条件，不允许无依据随机打印。
- 里程碑事件默认只触发一次。
- 趋势事件可以重复触发，但需要 cooldown，避免刷屏。
- 事件记录必须包含 evidence，说明触发依据，例如相关指标和阈值。

示例：

```text
事件：stable_replicator_lineage
条件：
- 某 lineage 连续多个窗口保持非零种群
- 该 lineage 的 population 占比超过阈值
- 平均 replication_accuracy 超过阈值
影响：
- 记录里程碑
- 更新 event_counters
证据：
- lineage_id
- population_share
- replication_accuracy
- window_ticks
```

未来事件系统扩展时，`cyanobacteria_like_lineage` 应表示“蓝藻样产氧光合原核谱系”，其条件应来自原核多样性、代谢复杂度、光合作用能力、水体覆盖率、阳光可用性、浅海面积等状态，而不是单独依赖氧气升高。该事件属于后续版本，不在 `v0.1` 实现。

## 7. 快照与日志

快照：

- 默认每 `1000` tick 保存一次普通快照。
- 始终维护 `saves/latest.json`。
- 快照必须包含完整状态、配置摘要、随机数状态和快照版本。
- 从快照恢复后，应继续演化，而不是重置 tick、seed 或随机源。

日志：

- `logs/history.log` 记录关键事件。
- `logs/metrics.csv` 记录周期性指标。
- 控制台默认只输出关键事件和阶段摘要，避免刷屏。
- 事件日志应包含 tick、事件名、可读消息和 evidence。

## 8. 测试计划

单元测试：

- 同一 seed 下运行结果可复现。
- RNA 复制会消耗资源并增加种群。
- 突变会创建新家族并保留父子谱系。
- 资源耗尽时种群增长受限。
- 稳定性低的家族更容易降解或灭绝。
- 事件系统只在条件满足时触发，并记录 evidence。
- 快照保存后恢复，继续运行的状态连续。

集成测试：

- 运行 `1000` tick 后生成日志、指标 CSV、latest 快照。
- 从 `latest.json` 恢复后继续运行，不重置 tick 和 seed。
- `inspect` 能打印当前世界摘要、优势 RNA 家族、最近事件。

Smoke test：

```text
python -m evogenesis run --ticks 1000
python -m evogenesis inspect saves/latest.json
python -m evogenesis run --resume saves/latest.json --ticks 100
```

## 9. 验收标准

- 项目可通过一条命令跑起来。
- 默认配置能看到 RNA 家族复制、突变、竞争、灭绝和优势谱系形成。
- 事件日志不是神谕式结论，而能说明事件发生依据。
- 快照可恢复，恢复后的 tick、seed、随机状态和种群状态连续。
- 配置可调，默认参数能产生有趣但不过度混乱的结果。
- 未引入未经说明的重量级依赖。
