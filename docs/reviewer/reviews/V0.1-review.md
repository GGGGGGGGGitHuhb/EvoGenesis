# EvoGenesis V0.1 Reviewer 审查口径

## 1. 审查目标

Reviewer 需要确认 Builder 完成的是 `v0.1 RNA World` 及 `v0.1.1 运行体验修正`，而不是提前实现后续生命史阶段。重点检查功能正确性、随机复现、快照连续性、事件解释性、中文输出、暂停/重置、异常路径和测试覆盖。

## 2. 范围检查

必须符合：

- 只实现 RNA 复制、突变、资源竞争、降解、谱系兴衰、事件日志和快照恢复。
- 事件系统可以具备可扩展结构，但注册事件应限于 RNA World。
- 不应实现前生命化学、原始细胞、原核生命、光合作用、氧气系统、蓝藻样谱系或宏观生态。
- `v0.1.1` 只允许改进运行体验和观察体验，不允许扩展生命阶段。

若发现超范围实现，应记录为风险，而不是替 Builder 修改。

## 3. 必测功能

命令行：

- `python -m evogenesis run --ticks 1000`
- `python -m evogenesis run --steps 1000`
- `python -m evogenesis run --until-tick 5000`
- `python -m evogenesis run --config configs/rna_world.default.json`
- `python -m evogenesis run --resume saves/latest.json --ticks 100`
- `python -m evogenesis pause`
- `python -m evogenesis reset`
- `python -m evogenesis inspect saves/latest.json`

状态与规则：

- 同一 seed 的结果可复现。
- 复制消耗资源并增加对应家族种群。
- 突变创建新家族，父子谱系字段可追踪。
- 资源枯竭能限制增长。
- 降解和灭绝不会产生负种群或悬空谱系。
- 统计指标与当前状态一致。

事件系统：

- 事件由条件触发，不是无依据随机打印。
- 里程碑事件只触发一次。
- 趋势事件如可重复触发，必须有 cooldown。
- 每条关键事件记录包含 tick、事件 ID、可读消息、evidence。
- evidence 能解释主要触发依据。
- 优势谱系事件必须列出具体谱系 ID、相关家族 ID、种群和占比。
- 灭绝事件必须列出本 tick 灭绝的家族 ID，数量过多时允许截断但要说明总数。

中文与总结：

- README、命令帮助、错误信息、运行摘要、事件消息和最终总结中文优先。
- 运行开始时应打印“RNA 世界开始演化”一类中文启动消息。
- 持续运行中应周期性打印关键参数，而不是只在结束时打印摘要。
- 暂停或运行结束时应生成中文总结，说明本次发生了什么、发展到哪一步、是否灭绝或资源崩溃。

快照：

- 快照包含版本、配置摘要、tick、seed、random_state、世界状态、RNA 家族、事件计数和最近事件。
- 恢复后继续运行不会重置 tick。
- 恢复后随机序列连续；相同路径的分段运行与一次性运行应一致或有明确设计说明。
- 损坏、缺字段或版本不兼容快照应被拒绝，并给出可读错误。
- 重置后下一次非恢复运行必须从 tick 0 开始。
- 同一输出目录不应允许多个模拟进程同时写 `latest.json`，除非显式指定独立输出目录。

## 4. 异常与边界

需要覆盖：

- ticks 为 0、负数、非数字。
- 配置文件不存在。
- 配置 JSON 格式错误。
- 配置参数越界，例如负资源、负突变率、复制准确率超过范围。
- 空种群、全体灭绝、资源为 0。
- 输出目录不存在时能创建，无法创建时能失败并说明原因。

## 5. 回归风险

重点观察：

- 模拟规则直接写日志，导致规则层与 IO 层耦合。
- 事件系统直接修改核心状态，导致事件与规则边界混乱。
- 快照没有保存随机源，导致恢复后不可复现。
- 指标 CSV 与实际状态不一致。
- 默认参数过于极端，导致总是立即灭绝或无限爆炸。
- 为后续版本提前加入过多字段，使 v0.1 状态难以理解。

## 6. Reviewer 交付要求

Reviewer 最终应产出工作报告，至少包含：

- 测试环境和命令。
- 通过项。
- 失败项与复现步骤。
- 风险等级和影响范围。
- 建议 Builder 返工的具体条目。
- 未覆盖测试和残余风险。

禁止在审查阶段替 Builder 修改实现代码。
