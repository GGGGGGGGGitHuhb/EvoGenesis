# EvoGenesis 概要设计

## 1. 项目定位

`EvoGenesis` 是一个命令行生物演化模拟器。项目从 RNA 世界起步，按生命演化顺序逐步扩展，最终成为地球模拟器中的生命模块。

首版聚焦 RNA 序列在原始环境中的复制、突变、资源竞争、降解、谱系兴衰、事件日志与快照恢复。整体体验应是“持续演化的科学观察”：用户启动模拟后，世界默认继续推进；用户可以暂停保存、重置测试数据、恢复上次状态，并观察关键参数与重大事件。

## 2. 技术路线

- Python 优先，使用标准库起步。
- 默认不引入重量级依赖；新增依赖必须说明必要性、替代方案和复杂度影响。
- 后续若出现明确性能瓶颈，仅迁移热点模块到 C++，不重写整体项目。
- 开发环境推荐 VS Code，配合 Python、Pylance、Ruff、Even Better TOML；如后续加入 C++，再引入 LLVM、CMake、Ninja 等工具链。
- 用户可见内容优先使用中文。英文标识可保留在代码、文件名和稳定事件 ID 中，但控制台、README、日志摘要和帮助文本应提供中文表达。

## 3. 总体架构原则

系统按职责分层，避免配置、状态、模拟规则、日志和快照相互硬编码。

- 命令入口层：负责命令解析、运行入口、用户输出。
- 模拟引擎层：负责主循环、tick 推进、模块调度、随机源管理。
- 领域状态层：负责世界状态、种群状态、谱系状态、统计指标。
- 规则模块层：负责具体演化规则，例如 RNA 复制、突变、竞争、降解。
- 事件系统层：负责从系统状态中识别里程碑、趋势和灾难事件，并生成可解释日志。
- 输入输出层：负责配置加载、事件日志、指标输出、快照保存和恢复。
- 运行控制层：负责暂停、重置、速度控制和运行会话边界，避免用户误用旧快照或测试数据。

## 4. 事件系统原则

重大事件不应是随机打印的固定文本，而应由系统状态触发。事件的基本模型是：

```text
事件 = 状态阈值 + 时间窗口 + 概率检定 + 影响结果 + 可解释日志
```

事件分三类：

- 里程碑事件：例如稳定复制子出现、原始细胞出现、蓝藻样谱系出现。
- 趋势事件：例如氧气持续上升、多样性下降、资源长期枯竭。
- 灾难事件：例如撞击、火山活动、雪球地球、大灭绝。

事件输出必须回答“发生了什么、为什么发生、影响了谁、后续状态怎样”。事件 ID 可以保持稳定英文标识，但用户可见名称和说明应使用中文。

## 5. 运行体验原则

演化模拟的默认心智模型应是“世界持续运行”，而不是“一次任务跑完就结束”。在仍以控制台为主的阶段，推荐采取分步演进：

- 短期：保留有限步运行用于测试，但明确参数含义；新增暂停保存、重置测试数据、中文输出和运行总结。
- 中期：支持持续运行模式，用户通过键盘中断或命令暂停并保存快照。
- 中后期：评估轻量图形界面，用面板显示主要参数、展开显示完整参数、主区域显示重要事件、侧栏显示事件发生时的证据状态。

## 6. 版本路线图

- `v0.1 RNA 世界基础版`：RNA 复制、突变、资源竞争、日志、快照。
- `v0.1.1 运行体验修正版`：持续运行语义、暂停保存、重置、速度控制、中文输出、运行总结、事件对象细化、tick 语义修正。
- `v0.2 前生命化学`：小分子池、核苷酸前体、聚合概率、环境催化、RNA 自发生成。
- `v0.3 原始细胞`：脂质膜泡、内部 RNA 池、膜稳定性、分裂、死亡。
- `v0.4 原核生命`：原核细胞、代谢类型、基因组、种群选择。
- `v0.5 光合作用与氧气`：光合作用、氧气积累、环境反馈、大氧化事件。
- `v0.6 真核细胞`：内共生、线粒体/叶绿体、复杂细胞、有性生殖。
- `v0.7 多细胞生命`：多细胞、细胞分化、组织复杂度、身体结构。
- `v0.8 生态系统`：物种、生态位、食物链、竞争、共生、疾病。
- `v0.9 宏观演化`：适应辐射、大灭绝、长期生命史。
- `v1.0 生命发生引擎`：完整命令行生物演化引擎，可作为未来地球模拟器的生命模块。

## 7. v0.1 与 v0.1.1 范围边界

`v0.1` 只实现 RNA 世界基础模拟。它可以为后续阶段预留概念接口，但不得提前实现前生命化学、原始细胞、原核生命、光合作用或宏观生态。

`v0.1.1` 不扩展生命史阶段，只修正运行体验和观察体验：

- 默认运行语义从“跑完 N tick 就结束”逐步转向“持续演化，可暂停”。
- `--ticks` 若保留，应明确表示“本次追加运行多少 tick”；如需要“运行到目标 tick”，应新增清晰参数，避免和恢复模式混淆。
- 重置必须能清理或隔离旧测试数据，使下一次从 tick 0 开始。
- 控制台与 README 应中文优先。
- 事件与总结要列出具体家族、谱系、关键参数和演化状态。

详细模块、状态字段、事件目录、测试项和验收标准见：

- `docs/builder/designs/V0.1-design.md`
- `docs/reviewer/reviews/V0.1-review.md`

## 8. 总体验收口径

- 可通过命令行启动一次完整模拟，并能从快照恢复。
- 默认配置能产生可读、有变化、可复现的 RNA 世界演化过程。
- 日志能解释关键事件发生的原因，而不是只给出结论。
- 控制台能持续显示关键参数，使用户看到“世界正在变化”。
- 快照能保存完整状态并恢复继续运行。
- 重置后能从 tick 0 开始，不误用旧版本或旧测试数据。
- 代码结构允许后续版本按路线图扩展，而不破坏 `v0.1` 的数据和命令入口。
